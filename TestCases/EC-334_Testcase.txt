Positive Test Cases

Test Case ID: EC-334-TC-001
Title: Add a product to the cart successfully
Test Type: Positive
Preconditions: User is already registered on the app

Steps:

1:
Action: Navigate to the application URL.
Expected result: Login page loads; username field, password field, and Login button are visible.

2:
Action: Enter "standard_user" in the Username field.
Expected result: Username field shows standard_user.

3:
Action: Enter "secret_sauce" in the Password field.
Expected result: Password field holds the value (masked).

4:
Action: Click the Login button.
Expected result: User is redirected to the inventory page.

5:
Action: Verify element visibility of product list with names, descriptions, prices, and **Add to Cart** buttons.
Expected result: User can see all product details with **Add to Cart** buttons.

6:
Action: Click **Add to Cart** for a specific product.
Expected result: Selected product is added to the cart; Button changes from **Add to Cart** to **Remove**; Cart icon badge updates with item count.

7:
Action: Click the Cart icon.
Expected result: User is navigated to the Cart page; added product(s) are displayed with correct details (name, price, quantity).

8:
Action: Add 5 products to the cart.
Expected result: Cart badge count reflects total items added as 5; added product continues to show **Remove** option on returning to Products page.

Expected Result: Product is successfully added to the cart and can be viewed in the Cart.
